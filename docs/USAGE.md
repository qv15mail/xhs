# 操作文档 — RedScope 使用指南

本指南介绍从环境启动到「采集 → 分析 → 仿写」完整使用流程。

## 一、环境准备

### 依赖要求
- Node.js 18+（推荐 20/22）
- Python 3.11+
- （可选）启用真实采集需要 Chromium：`python -m playwright install chromium`

### 启动后端
```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --port 8000
```
验证：浏览器或 curl 访问 `http://localhost:8000/api/health`，返回 `{"status":"ok"}`。

### 启动前端
```bash
cd frontend
npm install
npm run dev
```
访问 `http://localhost:5173`。

## 二、首次进入

1. 阅读弹出的「使用前须知」，勾选同意后进入。
2. 顶部右上角显示登录态徽标（绿点=在线 / 灰点=未登录）。

## 三、采集笔记

1. 进入「采集」页。
2. 右侧「登录态」卡片点击 **模拟扫码登录**（`real` 模式下为真实扫码）。
3. 左侧「新建采集任务」：
   - 输入**主题 / 关键词**（如：敏感肌护肤）；
   - 调整**采集数量**（滑块）、**排序**、是否**同时采集评论**；
4. 点击 **开始采集**。
5. 下方「采集任务」列表显示实时进度（进度条 + 已采/目标），完成后状态变为「已完成」。

> 默认 `mock` 模式会生成示例数据，便于快速体验全流程；切换 `real` 模式见下文。

## 四、查看笔记库

1. 进入「笔记库」。
2. 使用搜索框、排序（按互动量 / 发布时间）筛选。
3. 点击任意卡片，右侧抽屉展示正文、互动数据、话题标签与原文链接。
4. 抽屉内可一键跳转「拿去仿写」或「去拆解」。

## 五、内容分析

进入「分析」页，三个标签页：
- **选题热词**：热词云 + 词频明细（jieba 分词统计）。
- **爆款榜单**：按互动综合分排序，标注「数字标题 / 痛点开头 / 标签密集」等爆款特征。
- **内容拆解**：左侧选一篇笔记，右侧输出标题公式、开头钩子、正文骨架、标签策略。
  - 已配置 LLM 时由模型生成；否则使用内置启发式结果。

## 六、AI 仿写

1. 进入「仿写」页。
2. 左侧「创作设置」：
   - 可选**参考笔记**；
   - 输入**新主题 / 卖点**；
   - 选择**风格**（种草 / 测评 / 教程 / 情绪共鸣）与**长度**。
3. 点击 **生成仿写**。
4. 右侧得到：
   - **标题矩阵**（逐条可复制）；
   - **正文**（可直接编辑）；
   - **推荐话题标签**、**配图建议**。
5. 点击 **导出** 下载 Markdown，或逐项 **复制**。

## 七、设置

进入「设置」页：
- **LLM 配置**：填写 Base URL、API Key、模型、温度，点击「测试连接」验证。
- **采集配置**：默认采集数量、限速档位（推荐保守）、并发数、是否默认采集评论。
- **账号与数据**：清除登录态 / 清空本地数据。

修改后点击右上角 **保存设置**。

## 八、启用真实采集（real 模式）

> 真实采集涉及平台合规与风控，请务必使用本人账号、低频限速、仅采集公开内容。

1. 安装浏览器内核：
   ```bash
   cd backend && source .venv/bin/activate
   python -m playwright install chromium
   ```
2. 以 real 模式启动：
   ```bash
   REDSCOPE_COLLECT_MODE=real uvicorn app.main:app --port 8000
   ```
3. 首次会在 `REDSCOPE_USER_DATA_DIR`（默认 `./.xhs-user-data`）持久化登录态。

> 说明：`real` 模式保留了 Playwright 持久化上下文与限速骨架，**平台详情解析需按当前页面结构补全**（见 `output/xhs-delivery.md` 已知限制）。默认 `mock` 模式可稳定演示与测试。

## 九、常见问题

| 现象 | 排查 |
|------|------|
| 前端请求 404 / 跨域 | 确认后端已在 8000 启动；`REDSCOPE_CORS_ORIGINS` 是否含前端地址 |
| 采集一直 pending | 后端日志查看异常；mock 模式有限速延时属正常 |
| 拆解/仿写结果较模板化 | 未配置 LLM 时为启发式回退，配置真实 LLM 后质量提升 |
| real 模式仍返回示例数据 | 未安装 Chromium 或解析未补全，会回退 mock |

## 十、测试

```bash
# 后端单测
cd backend && source .venv/bin/activate && pytest

# 前端构建 + Lint
cd frontend && npm run build && npm run lint
```
