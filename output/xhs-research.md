# 小红书自动化采集分析与仿写工具 — 同类产品研究

> 阶段：research ｜ 工作模式：new ｜ 需求：输入主题 → 自动浏览/分析相关内容 → 仿写对应主题
> 本文档为后续 PRD / 架构 / UIUX 的约束来源。

## 1. 需求拆解

用户核心诉求是一个闭环工具：

1. **输入**：一个或多个主题 / 关键词（如「春季敏感肌护肤」「city walk 路线」）。
2. **采集**：自动浏览小红书相关笔记（列表 + 详情 + 互动数据 + 评论可选）。
3. **分析**：对采集到的内容做结构化分析（爆款规律、标题套路、选题方向、关键词、情绪、互动表现）。
4. **仿写**：基于分析产出同主题的高质量仿写笔记（标题 + 正文 + 话题标签 + 配图建议）。

## 2. 对标产品（5 类）

| 产品 | 类型 | 核心能力 | 对本项目的启发 |
|------|------|----------|----------------|
| **千瓜数据 (qian-gua)** | 商业数据分析平台 | 达人画像、热点笔记榜、品牌/行业分析、选题热词 | 数据看板 + 选题热词 + 笔记榜单的信息架构 |
| **新红 (newrank 系)** | 商业数据分析平台 | 笔记/达人搜索、内容趋势、品牌 PK | 关键词驱动的搜索→榜单→详情下钻路径 |
| **灰豚数据 (huitun)** | 商业数据分析平台 | 实时数据、平价/平替关键词捕捉、电商分析 | 实时性 + 关键词聚类的价值 |
| **MediaCrawler (开源)** | 多平台爬虫框架 | Playwright 登录态采集小红书笔记/评论，支持扫码/手机号/Cookie 登录，浏览器内执行签名 (x-s) | **采集层技术蓝本**：登录态 + 浏览器内签名 + 限速 |
| **XHSPlUS / AI 文案工具** | AI 创作辅助 | 爆款标题生成、爆款仿写、种草文案模板、关键词分析 | **仿写层蓝本**：拆解模板 + Prompt 工程 + 标题矩阵 |

补充：Apify 上有商用 RedNote/Xiaohongshu Scraper（按量付费 API）；OpenClaw/Skill 类项目尝试把小红书运营（选题/创作/复盘/复刻）做成 Agent 助手。

## 3. 共性功能（行业基线）

- 关键词 / 主题搜索，返回相关笔记列表。
- 笔记详情：标题、正文、图片、点赞/收藏/评论/分享数据、发布时间、作者。
- 榜单与排序：按互动量、增速、时间筛选「爆款」。
- 选题 / 热词分析：高频关键词、话题标签聚类。
- 内容拆解：标题套路、开头钩子、正文结构、emoji/标签使用。
- AI 创作：标题生成、正文仿写、种草模板、配图建议。
- 数据看板：趋势图、互动分布、Top 笔记卡片。

## 4. 关键流程（典型用户路径）

```
输入主题/关键词
  → 配置采集参数（数量、排序、是否含评论）
  → 自动采集（登录态浏览器 + 限速 + 去重）
  → 数据落库
  → 分析引擎（规则统计 + LLM 拆解）
  → 生成「选题洞察 + 爆款拆解」报告
  → 选择参考笔记 / 拆解模板
  → 仿写引擎（LLM）产出标题矩阵 + 正文 + 标签
  → 人工微调 → 导出
```

## 5. 采集层技术要点（来自 MediaCrawler 等方案）

- 小红书**官方 API 仅对认证品牌/MCN 开放**，普通开发者不可用 → 采用浏览器自动化方案。
- 主流方案：**Playwright 无头/有头浏览器 + 用户自有登录态（扫码/Cookie）**。
- 反爬关键：请求需要 **x-s / x-t 签名**，由前端 JS 生成；稳定做法是在**浏览器上下文内执行页面自带签名函数**，而非自行逆向。
- 必备工程手段：**请求限速、随机延时、并发控制、失败重试、去重、IP/账号风险隔离**。
- 风控现实：高频采集易触发验证码/封号，需「小号 + 低频 + 仅个人研究用途」。

## 6. 仿写层技术要点

- 仿写≠抄袭：采用「**爆款结构拆解 → 抽象为模板 → 注入新主题**」的 Prompt 工程范式。
- 拆解维度：标题公式（数字/痛点/悬念/对比）、开头钩子、正文骨架、情绪强度、emoji 与话题标签策略。
- 输出形态：标题矩阵（多个候选）、正文 1~N 版、推荐话题标签、配图/分镜建议。
- 模型：使用 OpenAI 兼容接口（可切换厂商/本地模型），将拆解结论作为 few-shot 与约束注入。

## 7. 差异化方向（本项目定位）

1. **一体化闭环**：采集→分析→仿写在同一本地工具内打通，无需在多个平台间切换。
2. **主题驱动的自动浏览**：用户只给主题，系统自动完成搜索、采集、筛选爆款。
3. **可解释的爆款拆解**：不仅给数据，还输出「为什么爆」的结构化模板，并直接驱动仿写。
4. **本地可控 + 自带登录态**：数据落本地库，账号与限速用户自管，降低合规与封号风险。
5. **人机协同仿写**：AI 出初稿 + 拆解依据，用户可一键重写/微调，保留创作主导权。

## 8. 合规与风险约束（硬约束，须写入 PRD/架构）

- 定位为**个人创作辅助 / 学习研究**用途，不做大规模商用数据贩卖。
- 采集需用户**使用自有账号登录**，默认**低频限速**，尊重 robots 与平台规则。
- 不存储/传播他人隐私；仿写产物须**原创化改写**，避免直接搬运侵权。
- 在产品内明确风险提示与免责声明；提供「仅采集公开数据 + 限速」默认值。

## 9. 推荐技术栈（待 PRD/架构细化）

- **采集/后端**：Python + FastAPI + Playwright（贴合 MediaCrawler 生态）+ SQLite（默认零依赖）。
- **分析/仿写**：规则统计 + OpenAI 兼容 LLM 客户端（可配置 base_url / model / key）。
- **前端**：React + TypeScript + Vite + Tailwind + shadcn/ui，图标用 **lucide-react**。
- **任务**：后端内置异步任务队列（asyncio）+ 任务进度轮询/SSE。
