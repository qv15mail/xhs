# PRD — 小红书自动化采集分析与仿写工具（代号 RedScope）

> 阶段：docs ｜ 依据：output/xhs-research.md ｜ 状态：待用户确认

## 1. 产品概述

RedScope 是一个**本地优先**的个人创作辅助工具：用户输入主题，系统自动采集小红书相关笔记、分析爆款规律，并基于拆解结论仿写出同主题的原创笔记。核心价值是把「选题 → 采集 → 拆解 → 创作」压缩进一个闭环。

- **目标用户**：内容创作者、运营、品牌种草、自媒体新人。
- **使用场景**：缺灵感时找选题、研究竞品爆款、批量产出初稿。
- **形态**：Web 应用（本地启动，桌面优先 + 响应式）。

## 2. 目标与非目标

**目标（v1）**
- 主题驱动的自动采集（搜索 + 列表 + 详情 + 互动数据）。
- 结构化分析：选题热词、爆款榜单、标题/正文拆解。
- AI 仿写：标题矩阵 + 正文 + 话题标签 + 配图建议。
- 数据看板与采集任务管理。

**非目标（v1 明确不做）**
- 不做自动发布 / 自动养号 / 自动引流。
- 不做大规模商用数据售卖、不绕过平台风控做高频采集。
- 不做多账号矩阵托管。

## 3. 合规红线（硬约束）

1. 采集必须使用**用户自有账号登录态**，默认**低频限速**（可配置，下限保守）。
2. 仅采集**公开可见**内容；不抓取私信、隐私信息。
3. 仿写产物定位为**二次原创**，产品内提示用户避免直接搬运。
4. 首次启动展示**风险与免责声明**，需用户确认后使用。

## 4. 用户故事

- US-1：作为创作者，我输入「露营装备」，能看到系统自动采集的相关笔记列表与互动数据。
- US-2：作为运营，我能看到该主题下的爆款榜单、高频热词、标题套路拆解。
- US-3：作为新人，我能基于一篇参考笔记，让 AI 仿写出 3 个标题 + 1 篇正文 + 推荐标签。
- US-4：作为用户，我能管理采集任务（查看进度、历史、重跑）。
- US-5：作为用户，我能配置 LLM（base_url / model / key）与采集参数（数量、排序、限速）。
- US-6：作为用户，我能把仿写结果复制 / 导出（Markdown）。

## 5. 功能需求

### F1 采集任务
- F1.1 输入主题/关键词、目标数量、排序（综合/最新/最热）、是否含评论。
- F1.2 登录态管理：扫码/Cookie 登录，登录状态可视化与失效提示。
- F1.3 任务执行：异步运行，实时进度（已采集/目标、当前状态、错误）。
- F1.4 限速与重试：随机延时、并发上限、失败重试、去重。
- F1.5 任务历史：列表、状态、重跑、删除。

### F2 数据与笔记库
- F2.1 笔记字段：标题、正文、图片 URL、作者、点赞/收藏/评论/分享、发布时间、链接。
- F2.2 笔记列表：筛选（互动量/时间）、排序、搜索、详情抽屉。
- F2.3 数据落本地库，支持按主题分组。

### F3 分析引擎
- F3.1 概览看板：笔记数、互动分布、Top 笔记卡片、采集趋势。
- F3.2 选题热词：关键词/话题标签词频与聚类（词云 + 列表）。
- F3.3 爆款榜单：按互动综合分排序，标注「为什么爆」标签。
- F3.4 内容拆解：对选定笔记输出标题公式、开头钩子、正文骨架、emoji/标签策略（规则 + LLM）。

### F4 仿写引擎
- F4.1 选择参考笔记或拆解模板 + 输入新主题/卖点。
- F4.2 生成：标题矩阵（≥3）、正文（可选风格/长度）、推荐话题标签、配图建议。
- F4.3 微调：一键重写、调整语气/长度、局部再生成。
- F4.4 导出：复制、导出 Markdown。

### F5 设置
- F5.1 LLM 配置：base_url、api_key、model、温度。
- F5.2 采集配置：默认数量、限速区间、并发、是否含评论。
- F5.3 登录态与数据管理：清除登录、清空数据。

## 6. 信息架构（页面）

1. **仪表盘 Dashboard**：全局概览 + 最近任务入口。
2. **采集 Collect**：新建任务 + 任务列表/进度。
3. **笔记库 Notes**：列表 + 筛选 + 详情。
4. **分析 Insights**：选题热词 + 爆款榜单 + 内容拆解。
5. **仿写 Compose**：参考选择 + 生成 + 微调 + 导出。
6. **设置 Settings**：LLM / 采集 / 账号 / 数据。

## 7. 验收标准（v1 DoD）

- 能用自有账号登录并对一个主题完成 ≥20 条笔记采集且落库。
- 分析页能展示热词、爆款榜单、对单篇笔记的拆解。
- 仿写页能基于参考产出标题矩阵 + 正文 + 标签，并可导出 Markdown。
- 限速、重试、任务进度、错误提示可用。
- 前端可运行、可演示、无明显报错；关键交互有 loading/empty/error 态。

## 8. 指标（参考）

- 单主题采集成功率、平均耗时；仿写采纳率（用户复制/导出占比）；任务失败率。

## 9. 里程碑

- M1 前端可演示（含 mock 数据）：页面骨架 + 交互 + 看板。
- M2 后端采集 + 落库打通。
- M3 分析 + 仿写引擎接入 LLM。
- M4 质量门禁 + 交付清单。

## 10. 风险

- 平台风控/封号 → 限速 + 自有小号 + 个人用途。
- 签名/反爬变化 → 采用浏览器内签名而非逆向，降低维护成本。
- LLM 成本/可用性 → 可配置厂商与模型，支持本地兼容接口。
